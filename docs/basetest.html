BaseTest

global abstract class BaseTest extends BaseExplicitWaitTest
Description BaseTest is an abstract base class that defines a single test in Cinnamon.

When you run a Cinnamon test, the Cinnamon framework manages the entire test lifecycle and executes the setup, test, and teardown methods during test execution. The Context context parameter must be used to store and retrieve the test fixture throughout the test. 


   public class TestAccountPage extends BaseTest {
       // Create an Account record as a test fixture
       public override void setUp(Context context) {
           Account acc = new Account();
           acc.name = 'account_' + System.currentTimeMillis();
           acc.site = 'San Francisco';
           insert acc;
           context.put('accId', acc.Id);
       }
       // Set the URL for the page to start from for this test
       public override String getStartingPath(Context context) {
           String accId = (String) context.get('accId');
           return '/apex/EditAccount?id=' + accId;
       }
       // All test logic should be in this method
       public override void test(Context context) {
           String accId = (String) context.get('accId');
           AccountDetailPageObject accountPage = (AccountDetailPageObject) context.getPageObject(AccountDetailPageObject.class);
           accountPage.typeAccountSiteField('New York');
           accountPage.clickSave();
           Account acc = [select Id, name, site from Account where Id = :accId];
           System.assert(acc.site.equalsIgnoreCase('New York'), 'site does not match');
       }
       // Clean up the account record at the end of test execution
       public override void tearDown(Context context) {
           List Ids = new List();
           Ids.add((String) context.get('accId'));
           Database.delete(Ids);
       }
   }

global override virtual ExpectedCondition getStartingPathExpectedCondition(context.html'>Context context)
Description Returns an expected condition that is evaluated during the test start up after the URL request from BaseTest.getStartingPath(). The base implemenation is return ExpectedConditions.visibilityOf(new CSSLocator('body.sfdcBody')); This implementation works well for Visualforce pages that have simple page load events. Tests for more complex pages must override this method and provide a more specific condition to meet before the test starts.
