ExpectedConditions

global with sharing class ExpectedConditions
DescriptionExpectedConditions is a utility class that provides a list of canned ExpectedConditions to use when a test needs to wait for a certain condition to become true or false. ExpectedConditions are modeled after the Selenium Webdriver ExpectedConditions class.

For example, the sample code below uses a method from ExpectedCondition to wait for an HTML element containing a string to render after clicking the Save button. Once the element appears, the ExpectedCondition is satisfied and the test completes. 


  public class TestWithExpectedCondition extends cinnamon.BaseTest {
    private CustomPageObject customPage;
      public override void setUp(cinnamon.Context context) {
      customPage = (CustomPageObject) context.getPageObject(CustomPageObject.class);
      customPage.initializePageObject();
    }
    public override void test(cinnamon.Context context) {
      cinnamon.ElementLocator locator = new cinnamon.ElementLocator('div.title');
      customPage.fillForm();
      customPage.clickSave();
      // this condition waits until div.title element that contains the text 'Hello World' appears
      cinnamon.ExpectedCondition condition = cinnamon.ExpectedConditions.textToBePresentInElement(locator, 'Hello World!');
      customPage.getWait().until(condition);
    }
  }
