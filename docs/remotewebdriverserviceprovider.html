global interface RemoteWebDriverServiceProvider

Methods

  getAvailableBrowsers

List<String> getAvailableBrowsers()
DescriptionImplementors return a list of strings representing the supported OS/browser combinations. These strings allow the tester to specify the environment. This list does not contain duplicate entries.
ReturnsThe list of os-browser combination keys the provider supports (e.g. "Win 7 IE 9," "IOS8 iPad")

  getCapabilitiesFor

Capabilities getCapabilitiesFor(String environmentKey)
DescriptionImplementors return a Capabilities instance with the minimal set of required capabilities (browserName, browserVersion, platformName, platformVersion). The Capabilities class provides a constructor with these parameters for convenience. This Capabilities instance is sent to the remote WebDriver service as the 'desiredCapabilities' parameter.
Returnsinstance of Capabilities with OS and browser capabilities

  getRemoteWebDriverServiceUrl

String getRemoteWebDriverServiceUrl()
DescriptionImplementors return the URL used to connect to the remote WebDriver server. Define the necessary protocol and any special authentication mechanism in this method. For example, this method could return something like: http://myuser:myaccesskey@some.webdriver.org:4444/wd/hub.
ReturnsThe URL to the remote WebDriver server including protocol, authentication, and path.

  getSettingsPage

PageReference getSettingsPage()
DescriptionServiceProviders may need additional settings for full functionality. Implementors must proide a Visualforce page where these additional settings can be configured. It is the implementor's responsibility to store this information in the format it deems appropriate.
ReturnsA Visualforce page reference to use in the settings page. This page loads inside an iframe with a maximum height of 200px.
